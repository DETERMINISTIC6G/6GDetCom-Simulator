d6g.apps.edgecloud

Name Type Description
UdpEdgeCloudApp simple module

The module is designed to simulate a UDP-based application in edge cloud scenarios. This module builds upon the UdpEchoApp, inheriting its functionalities and adding the capability to introduce specific delays. Such delays might be used to simulate various network conditions or service latencies in edge cloud settings. Users can specify destination addresses, destination ports, and custom delays to simulate application behavior in edge environments.

d6g.devices.sixg

Name Type Description
DetCom compound module

DetCom extends the DetComAdapter module. This module depicts a simulated network device involved in communication. It defines specific parameters which provide bridging mechanism.
DetComAdapter compound module

The module extends the TsnSwitch module to offer advanced Ethernet communication capabilities. It provides a variety of configurable parameters to simulate different networking behaviors at both the MAC and PHY layers. These parameters include options for frame preemption, egress traffic shaping, and cut-through switching, thus allowing a flexible setup to emulate real-world networking scenarios.
NNIChannel channel

This module represents a Ethernet communication channel with specific attributes such as channel length and data rate. Extending the capabilities of the 'EthernetLink' module from INET ,it provides a enhanced framework for network simulations

UE compound module

This module reperesents a User Equipment(UE) and extends the behavior of 'LayeredEthernetInterface' module of INET. The module configures parameters related to the physical layer's inter frame gap inserter duration and the communication bitrate specific to UE.
UnitWithAdapter compound module

This module provides a network communication scenarios based on whether the communication is wireless. It configures the necessary submodules ,connections,and gates to facilitate dynamic communication pattern.

d6g.distribution.histogram

Name Type Description
Histogram simple module

This module groups the delays based on the bins assigned as per histogram xml. It reads histogram configuration, calculates accumulated bin counts, generates random values within bins, and implements binary search for bin selection.

DetComNetworkBase network

The network is a specialized network type designed for deterministic communications. It extends TsnNetworkBase, a network designed for Time-Sensitive Networking.

d6g.pairwise

Name Type Description
BridgingLayerWithPairwiseDelay compound module

This module provides various layer 2 services such as packet forwarding, interface selection, virtual LAN handling, stream handling.
PairwiseDelayer simple module

The module is designed to delay packets based on pairwise configurations. This module works by accepting packets and applying a delay according to the specified delayConfig. This can be particularly useful in scenarios where varying delays are required for different packet pairs or flows. It's an extension of PacketDelayerBase and adheres to the IPacketDelayer contract. Users can provide custom delay configurations using the delayConfig XML parameter.

d6g.simulations

Name Type Description
TsnTestNetwork network

An extension of the TsnNetworkBase from the INET framework. The network consists of a client, a switch, and a server. A Histogram submodule is also included for statistical analysis, with data sourced from example-hist.xml.

d6g.simulations.baseline

Name Type Description
Baseline network

The network is to evaluate the data communication from a talker (sender) through multiple bridges and adapters to a listener (receiver). It Involves both Time-Sensitive Networking (TSN) and Deterministic Communication (DetCom) elements.

d6g.simulations.cooperativemanufacturing

Name Type Description
AGV compound module

The module extends UnitWithAdapter to represent an AGV in a cooperative manufacturing environment. It has the ability to communicate wirelessly and has two Ethernet submodules: one for communication between AGVs (interAgv) and another for communication to a processing cell (interToCell).
CooperativeManufacturing network

This network focuses on a cooperative manufacturing environment. The network simulates communication between AGVs (Automated Guided Vehicles) and a processing cell via a DetCom (Deterministic Communication) module to ensure reliable and timely data transmission.
ProcessingCell compound module

The module is designed to simulate a processing unit in a cooperative manufacturing environment. It extends UnitWithAdapter, adding specific configurations like Ethernet bitrate and submodules for controlling swarm operations and status.
